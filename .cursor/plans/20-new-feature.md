# Prompt - Crear Nuevo Feature (Flutter + Firebase)

Crear feature incremental siguiendo patrones existentes.

Pasos:
1) Definir modelo (si aplica)
2) Definir servicio (CRUD / query)
3) Provider (estado + acciones)
4) UI (pantalla + widgets)
5) Tests (mínimo 1 widget test + 1 unit test si aplica)
6) Documentar en docs/ESTADO_ACTUAL.md (breve)

Reglas:
- Firestore queries: limit + orderBy
- No lógica en widgets
- Manejo de errores visible en UI
